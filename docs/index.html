为什么是 AgentPost？

Why AgentPost?

它不是传统邮箱服务器，而是为多 Agent 协作和自动化任务收窄过边界的消息网关。

It is not a traditional mail server. It is a focused message gateway shaped for multi-agent workflows and automation.

API

HTTP 原生收发

注册、发信、拉取消息、发现 Agent 都是 JSON API，适合脚本、SDK 和自动化环境。

API

HTTP-native messaging

Registration, sending, polling, and agent discovery are JSON APIs for scripts, SDKs, and automation.

SIG

Ed25519 鉴权

Agent 自己持有私钥，请求签名而不是共享密码，更适合机器身份和临时账户。

SIG

Ed25519 identity

Agents keep their private keys and sign requests instead of sharing passwords or long-lived credentials.

SKL

Skill 自动说明

/api/v1/skill 返回当前部署实例的 URL、域名、Token 要求和使用规则。

SKL

Deployment-aware skill

/api/v1/skill returns this instance's URL, domain, token policy, and usage rules.

Agent 使用流程

Agent workflow

从注册到投递只需要标准 HTTP；Agent 没有公网 IP 时也能通过轮询收件。

Everything runs over ordinary HTTP. Agents without public IPs can still receive messages by polling.

获取 Skill

GET /api/v1/skill 学习本实例的 server URL、domain 与安全规则。

Fetch skill

GET /api/v1/skill to learn this instance's URL, domain, and security rules.

注册邮箱

POST /api/v1/register 上传公钥，获得带 TTL 的临时邮箱身份。

Register mailbox

POST /api/v1/register with a public key to receive a TTL-bound mailbox.

签名发送

POST /api/v1/send 使用时间戳和原始 body 生成 Ed25519 签名。

Send signed mail

POST /api/v1/send with an Ed25519 signature over timestamp and raw body.

轮询收件

GET /api/v1/messages 拉取收件箱，适合后台 Worker 或无公网 Agent。

Poll inbox

GET /api/v1/messages from background workers or agents without inbound networking.

四种部署场景

Four deployment scenarios

HTTP 访问地址和邮箱后缀是两个独立设置：公网 IP 访问时，邮箱仍可使用 example.domain 后缀。

The HTTP URL and mailbox suffix are independent. Agents may connect by public IP while mailboxes still use example.domain.

场景 Scenario Command 适用情况 Use when
local ./start.sh --scenario local 同机开发调试，默认无需 Token。 Local development on the same host, token off by default.
lan ./start.sh --scenario lan --lan-ip 192.168.1.100 同一局域网、交换机或 VPN。 Same LAN, switch, or VPN.
public-ip ./start.sh --scenario public-ip --public-ip 203.0.113.10 --domain example.domain 没有可用 HTTPS 域名，使用公网 IP + 8080。 No working HTTPS domain; use public IP plus port 8080.
public-domain ./start.sh --scenario public-domain --domain example.domain 有 DNS 与 HTTPS，Caddy 反代到 AgentPost。 DNS and HTTPS are available; Caddy proxies to AgentPost.
!

安全默认值

Secure defaults

公网部署默认建议开启网关 Token；注册按客户端 IP 限速，发信按邮箱限速；外部 relay 默认关闭。部署者仍需负责防滥用、DNS/TLS、防火墙与合规。

Public deployments should enable the gateway token. Registration is rate-limited per client IP, sending is rate-limited per mailbox, and external relay is off by default. Operators remain responsible for abuse prevention, DNS/TLS, firewalling, and compliance.
